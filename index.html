Sistemas Numéricos

Los 4 sistemas fundamentales en computación

2

Binario

Base 2: 0, 1

• Cada dígito es 1 bit

• Estados: encendido/apagado

• Ejemplo: 1011₂ = 11₁₀

• Uso: CPUs, RAM, transistores

8

Octal

Base 8: 0-7

• Agrupa 3 bits binarios

• 3 bits = 1 dígito octal

• Ejemplo: 17₈ = 15₁₀

• Uso: Permisos Unix/Linux

10

Decimal

Base 10: 0-9

• Sistema humano cotidiano

• Base posicional: 10ⁿ

• Referencia universal

• Uso: Vida diaria, contabilidad

16

Hexadecimal

Base 16: 0-F

• Agrupa 4 bits binarios

• 4 bits = 1 dígito hex

• Ejemplo: FF₁₆ = 255₁₀

• Uso: Colores web, memoria RAM

Valor = Σ dᵢ × Bⁱ   (donde B = base, dᵢ = dígito, i = posición)

📐 Métodos de Conversión entre Sistemas

🔢 Decimal → Binario  (División sucesiva por 2)

Ej: 45₁₀ → Binario
45 ÷ 2 = 22  r 1 ⟵ bit menos sig.
22 ÷ 2 = 11  r 0
11 ÷ 2 =  5  r 1
 5 ÷ 2 =  2  r 1
 2 ÷ 2 =  1  r 0
 1 ÷ 2 =  0  r 1 ⟵ bit más sig.

Leer residuos ↑: 101101₂
Verif: 32+8+4+1 = 45 ✓

🔢 Decimal → Octal  (División sucesiva por 8)

Ej: 156₁₀ → Octal
156 ÷ 8 = 19  r 4 ⟵ dígito menos sig.
 19 ÷ 8 =  2  r 3
  2 ÷ 8 =  0  r 2 ⟵ dígito más sig.

Leer residuos ↑: 234₈
Verif: 2×64 + 3×8 + 4 = 156 ✓

🔗 Binario ↔ Octal  (Grupos de 3 bits)

Ej: 101101₂ → Octal
Agrupar de derecha a izquierda:
 101 | 101
  ↓     ↓
  5      5
Resultado: 55₈

Octal → Binario: cada dígito → 3 bits
7₈ = 111₂  |  5₈ = 101₂

🔗 Binario ↔ Hexadecimal  (Grupos de 4 bits)

Ej: 10101101₂ → Hex
Agrupar de derecha a izquierda:
1010 | 1101
  ↓      ↓
  A      D
Resultado: AD₁₆

Dec → Hex (÷16):
191÷16=11 r15(F) → 11÷16=0 r11(B) → BF₁₆

🎯
¿Por qué Octal?

Los sistemas Unix usan octal para permisos de archivos (rwx = 3 bits). Cada dígito octal representa exactamente 3 bits, facilitando la lectura de permisos binarios.

🎨
¿Por qué Hexadecimal?

Un byte (8 bits) = 2 dígitos hex. Usado en colores web (#FF5733), direcciones de memoria (0x7FFF), y depuración de registros CPU. Compacto y legible.

⚡
Binario como Puente

Para convertir Octal ↔ Hexadecimal, siempre usa Binario como intermediario. Octal → Binario (3 bits) → Hexadecimal (4 bits agrupados).

Ej. 7 — 1101001₂ → Decimal

Pos:  6  5  4  3  2  1  0
Bit:  1  1  0  1  0  0  1

1×2⁵ = 32
0×2⁴ =  0
1×2³ =  8
0×2² =  0  |  0×2¹ = 0  |  1×2⁰ = 1

0×2³=0 + 1×2²=4 + 1×2¹=2 + 0

🏗️

Arquitectura Interna

BIU — Bus Interface Unit
• Prefetch: cola 6 bytes
• Calcula dir. física: Seg×16+Off
• Gestiona buses externos
EU — Execution Unit
• ALU 16 bits
• Decodifica instrucciones
• Maneja registros y flags
• Opera en paralelo con BIU
Pipeline 2 etapas
↑ rendimiento ~50%
🗺️

Mapa de Memoria (1 MB)

00000-003FF → Tabla vectores INT
00400-9FFFF → RAM usuario (~640 KB)
A0000-BFFFF → Video (VGA)
C0000-EFFFF → ROM expansión
FFFF0-FFFFF → ROM BIOS
Dir. Física = CS × 16 + IP

Modos de Direccionamiento del 8086

Formas de especificar la ubicación de operandos en memoria

⚡
Inmediato
MOV AX, 25

El operando es una constante incluida en la instrucción. No requiere acceso a memoria. Rápido pero limitado a valores fijos.

📝
Por Registro
MOV AX, BX

Ambos operandos están en registros internos. Es el modo más rápido (sin acceso a memoria). Limitado por el número de registros disponibles.

🎯
Directo
MOV AX, [1000h]

La dirección efectiva es una constante. Se accede a DS:1000h. Útil para variables globales. Limitado a offset de 16 bits.

🔗
Indirecto por Registro
MOV AX, [BX]

La dirección está en un registro base (BX, BP) o índice (SI, DI). Permite acceso dinámico a arreglos y estructuras de datos.

📊
Indexado con Desplazamiento
MOV AX, [SI + 10]

Dirección = Registro índice (SI/DI) + constante. Ideal para acceder a campos de estructuras o arreglos con offset fijo.

🧮
Base + Índice + Desplazamiento
MOV AX, [BX + SI + 5]

Modo más flexible: Base (BX/BP) + Índice (SI/DI) + Constante 8/16 bits. Perfecto para matrices bidimensionales y registros de activación.

Sistema de Interrupciones

Mecanismo de atención a eventos externos e internos

🔔 Tipos de Interrupciones

H
Hardware (IRQ)
Señales eléctricas de dispositivos externos. Ej: teclado, timer, disco. Gestionadas por PIC 8259.
S
Software (INT)
Generadas por instrucción INT. Usadas para llamadas al sistema operativo (DOS: INT 21h).
I
Internas
Division by zero, overflow, breakpoint (INT 3), paso a paso (Trap Flag).

📋 Vector de Interrupciones

Tabla en RAM (00000h-003FFh) con 256 entradas de 4 bytes cada una:

Offset 0: IP de manejador
Offset 2: CS de manejador

Ejemplo INT 21h:
Dirección = 21h × 4 = 84h = 0000:0084
[0084] = IP, [0086] = CS
